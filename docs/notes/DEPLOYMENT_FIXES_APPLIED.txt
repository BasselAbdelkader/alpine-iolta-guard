ISSUE #1: Resource Limits Too High
-----------------------------------
ERROR MESSAGE:
  "range of CPUs is from 0.01 to 1.00, as there are only 1 CPUs available"

ROOT CAUSE:
  docker-compose.alpine.yml had resource limits configured for production servers:
  - Database: reservations.cpus: '0.5' (needs at least 1 CPU available)
  - Backend: reservations.cpus: '0.5'
  - Frontend: reservations.cpus: '0.25'
  
  Development machine only has 1 CPU, so reserving 0.5 fails.

FIX APPLIED:
  Commented out all resource limits in docker-compose.alpine.yml:
  - Lines 47-54 (database)
  - Lines 111-118 (backend)
  - Lines 153-160 (frontend)

BACKUP CREATED:
  docker-compose.alpine.yml.backup_before_resource_fix

---

ISSUE #2: Frontend Permission Denied
-------------------------------------
ERROR MESSAGE:
  nginx: [emerg] open() "/run/nginx.pid" failed (13: Permission denied)

ROOT CAUSE:
  Dockerfile.alpine.frontend runs nginx as non-root user (nginx)
  But /run directory was owned by root
  nginx user couldn't create pid file

FIX APPLIED:
  Added to Dockerfile.alpine.frontend (lines 50-52):
  RUN mkdir -p /var/run /run && \
      chown -R nginx:nginx /var/run /run

RESULT:
  Frontend now starts successfully as nginx user (secure)
