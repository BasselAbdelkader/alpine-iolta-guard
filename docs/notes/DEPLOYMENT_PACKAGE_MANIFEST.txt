================================================================================
WHAT HAPPENS DURING DEPLOYMENT
================================================================================

Step 1: docker-compose build
  - Pulls base images (python:3.11-alpine, nginx:alpine, postgres:16-alpine)
  - Installs Python packages (Django, DRF, psycopg2, etc.)
  - Copies backend/ folder (includes migrations)
  - Copies frontend/ folder (HTML/CSS/JS)
  - Creates container images
  Duration: ~2 minutes

Step 2: docker-compose up -d
  - Creates Docker network (iolta_network)
  - Creates Docker volumes (db_data, static_files, media_files)
  - Starts PostgreSQL container
  - Starts Django backend container
  - Starts Nginx frontend container
  Duration: ~20 seconds

Step 3: Wait for healthy
  - PostgreSQL initializes database
  - Django backend connects to database
  - Nginx frontend starts serving
  Duration: ~2 minutes

Step 4: python manage.py migrate
  - Creates django_migrations table
  - Runs clients/0001_initial.py (creates clients, cases tables)
  - Runs vendors/0001_initial.py (creates vendors, vendor_types tables)
  - Runs bank_accounts/0001_initial.py (creates 4 tables)
  - Runs settlements/0001_initial.py (creates 3 tables)
  - Runs settings/0001_initial.py (creates 4 tables)
  - Runs reports/0001_initial.py (creates views)
  - Creates all indexes and constraints
  Duration: <1 second

Step 5: Import data (optional)
  - Reads essential_data_only.sql
  - Inserts 289 records across 7 tables
  - Verifies foreign key relationships
  Duration: ~1 second

Step 6: Create superuser
  - Prompts for username, email, password
  - Creates admin user in auth_user table
  - Grants superuser privileges
  Duration: ~30 seconds (user input)

Step 7: Verification
  - Checks container status
  - Tests web interface
  - Counts database records
  Duration: ~5 seconds

Total: ~5-6 minutes

================================================================================
TROUBLESHOOTING
================================================================================

If docker-compose build fails:
  - Check Docker is running: docker ps
  - Check disk space: df -h
  - Check logs: docker-compose build 2>&1 | tee build.log

If containers don't start:
  - Check port conflicts: netstat -tlnp | grep :80
  - Check logs: docker-compose logs
  - Check .env file: cat .env

If migrations fail:
  - Check backend logs: docker-compose logs backend
  - Check database is up: docker exec iolta_db_alpine pg_isready
  - Check migration files exist: ls backend/apps/*/migrations/0001_initial.py

If data import fails:
  - Check SQL file exists: ls -lh essential_data_only.sql
  - Check database is ready: docker exec iolta_db_alpine psql -U iolta_user -d iolta_guard_db -c "\dt"
  - Check for errors: docker-compose logs database

If web interface doesn't load:
  - Check frontend logs: docker-compose logs frontend
  - Check backend is up: curl http://localhost:8000/api/v1/health/
  - Check firewall: sudo ufw status
