## Understanding Clients vs Cases

### What is a Client?

A **client** is a person or company that your law firm represents.

**Example:**
- John Smith (person)
- ABC Corporation (company)
- Mary Johnson (person)

**One client can have multiple cases.**

### What is a Case?

A **case** is a specific legal matter for a client.

**Example:**
John Smith might have:
- Case #2025-001: Personal Injury Settlement
- Case #2025-045: Real Estate Closing

**Each case has its own separate trust account balance.**

### Why Separate Them?

This separation helps you:
- Track money for each specific legal matter
- Keep different cases' funds separate
- Generate accurate reports per case
- Comply with attorney trust account rules

## Working with Clients

### Viewing All Clients

**Steps:**
1. Click "**Clients**" in the main menu
2. You'll see a list of all clients

**Screenshot placeholder:** [URL_HERE - Clients List Page]

**What you'll see:**
- Client name
- Email address
- Phone number
- Total balance (all cases combined)
- Status (Active/Inactive)

### Searching for a Client

**There are three ways to find a client:**

**Method 1: Search Box**
1. Look for the search box at the top of the client list
2. Type the client's name (first or last)
3. Results appear as you type

**Method 2: Sort the List**
1. Click on column headers to sort
2. Click "Name" to sort alphabetically
3. Click "Balance" to sort by amount

**Method 3: Filter**
1. Use the filter options (if available)
2. Filter by status: Active or Inactive
3. Filter by balance: With funds, Zero balance, etc.

### Adding a New Client

**Steps:**
1. Click "Clients" menu
2. Click "**+ Add New Client**" button
3. Fill in the form:

**Required Fields:**
- First Name
- Last Name

**Optional but Recommended Fields:**
- Email
- Phone Number
- Street Address
- City
- State (dropdown menu)
- Zip Code

4. Click "**Save**"

**Screenshot placeholder:** [URL_HERE - Add Client Form]

**Tips:**
- ✅ **DO** enter complete information (helps avoid duplicates)
- ✅ **DO** use proper capitalization (John Smith, not john smith)
- ✅ **DO** double-check spelling
- ❌ **DON'T** create duplicate clients (search first!)

### Viewing Client Details

**Steps:**
1. Go to Clients page
2. Click on the client's name
3. You'll see the Client Detail page

**Screenshot placeholder:** [URL_HERE - Client Detail Page]

**What's on the Client Detail Page:**

**Section 1: Client Information**
- Name, email, phone

- Total balance across all cases
- Status (Active/Inactive)

**Section 2: Cases List**
- All cases for this client

- Edit Client

### Editing Client Information

**Steps:**
1. Go to Client Detail page
2. Click "**Edit Client**" button
3. Modify the information
4. Click "**Save**"

**Screenshot placeholder:** [URL_HERE - Edit Client Form]

**What You Can Edit:**
- ✅ Email, phone, address (contact info)
- ✅ Status (Active/Inactive)
- ❌ Client name (should not be changed - see note below)

**Important Note About Editing Names:**
If you need to change a client's name, contact your supervisor. Name changes affect records and reports.

### Deleting a Client

**Can I delete a client?**

**Generally, NO.** You should not delete clients because:
- Audit requirements need complete history
- Transactions would be lost
- Reports would be inaccurate

**Instead:** Mark the client as "Inactive"

**Steps to Deactivate:**
1. Go to Client Detail page
2. Click "Edit Client"
3. Change status to "Inactive"
4. Click "Save"

**Inactive clients:**
- Don't appear in normal lists (use filter to see them)
- Cannot have new transactions
- Retain all history
- Can be reactivated if needed

### Client Status Explained

**Active with Funds:**
- Has money in trust account
- Can accept new transactions
- Shows in main client list

**Active - Zero Balance:**
- No money in trust account currently
- Can accept new transactions
- Shows in main client list

**Inactive:**
- Case closed, funds disbursed
- Cannot accept new transactions
- Hidden from main list (use filter to view)

## Working with Cases

### Viewing Cases

**Option 1: From Client Detail Page**
1. Go to client detail page
2. Scroll to "Cases" section
3. All cases for this client are listed

**Option 2: Search All Cases**
(If your system has a Cases menu)
1. Click "Cases" in menu
2. View all cases across all clients

### Adding a New Case

**Steps:**
1. Go to Client Detail page
2. Click "**+ Add New Case**" button
3. Fill in the form:

**Required Fields:**
- Case Number (must be unique)
- Case Title
- Opened Date

**Optional Fields:**
- Matter Type (Personal Injury, Real Estate, etc.)
- Description
- Opposing Party
- Court/Jurisdiction

4. Click "**Save**"

**Screenshot placeholder:** [URL_HERE - Add Case Form]

**Case Number Guidelines:**
- Use your firm's numbering system
- Common formats:
  - YYYY-NNN (2025-001, 2025-002, etc.)
  - ClientInitials-YYYY-NN (JS-2025-01)
  - Attorney-YYYY-NN (MA-2025-15)
- Once created, case number cannot be changed
- Must be unique across all cases

### Viewing Case Details

**Steps:**
1. From Client Detail page, click on a case name
2. OR navigate directly to case if you know the case number

**Screenshot placeholder:** [URL_HERE - Case Detail Page]

**What's on the Case Detail Page:**

**Section 1: Case Information**
- Case number
- Case title
- Client name (clickable to go back to client)
- Matter type
- Opened date
- Status (Open/Closed)
- Closed date (if closed)
- Current balance

**Section 2: Transactions**
- List of all deposits and withdrawals
- Date, type, amount, description

**Section 3: Action Buttons**
- Add Transaction
- Edit Case
- Close Case
- Print Case Report

### Opening vs Closing Cases

**Open Case:**
- Active legal matter
- Can add transactions
- Shows in active cases list

**Closed Case:**
- Legal matter completed
- **Cannot add new transactions** (system prevents this)
- Retains all history
- Shows in closed cases list

**When to Close a Case:**
- Legal matter is resolved
- All funds have been disbursed
- Balance is zero
- Final accounting is complete

**How to Close a Case:**
1. Ensure balance is zero (or explain remaining balance)
2. Disburse all remaining funds
3. Go to Case Detail page
4. Click "Edit Case" or "Close Case"
5. Change status to "Closed"
6. Enter Closed Date
7. Click "Save"

**Screenshot placeholder:** [URL_HERE - Close Case]

**Important:** Once closed, you cannot add transactions. If you need to add a transaction, contact your supervisor to reopen the case.

## Best Practices

### For Clients

✅ **DO:**
- Search before adding new clients (avoid duplicates)
- Enter complete contact information
- Use proper capitalization and spelling
- Keep email addresses current
- Mark clients inactive instead of deleting

❌ **DON'T:**
- Create duplicate client records
- Use abbreviations in names (unless part of legal name)
- Delete clients with transaction history

### For Cases

✅ **DO:**
- Use consistent case numbering format
- Give cases descriptive titles
- Close cases when matters are resolved
- Ensure zero balance before closing
- Add descriptions for clarity

❌ **DON'T:**
- Use duplicate case numbers
- Leave cases open after completion
- Close cases with remaining balance (unless documented)
- Reuse case numbers

### Data Entry Tips

1. **Be Accurate:** Double-check names and numbers
2. **Be Consistent:** Use the same format every time
3. **Be Complete:** Fill in all available information
4. **Be Clear:** Use descriptive titles and descriptions
5. **Ask Questions:** If unsure, ask your supervisor

## Common Questions

**Q: Can one client have multiple cases?**
A: Yes! Clients often have multiple legal matters.

**Q: Can I move a case to a different client?**
A: No, cases cannot be moved. If this is needed, contact your supervisor or IT support.

**Q: What happens if I create a duplicate client by mistake?**
A: Contact your supervisor immediately. They can help merge or deactivate the duplicate.

**Q: Can I change a case number after creation?**
A: No, case numbers are permanent. This ensures transaction history integrity.

**Q: What if a closed case needs to be reopened?**
A: Contact your supervisor. They can reopen cases if necessary.

**Q: How long should we keep inactive clients in the system?**
A: Keep them permanently for audit trail purposes. They don't affect system performance.
